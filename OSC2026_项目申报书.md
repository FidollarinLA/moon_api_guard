## 项目简介

`moon_api_guard` 是一个面向 MoonBit 生态的公共 API 兼容性守卫工具。项目基于 `moon info` 生成的 `.mbti` 接口描述，对比旧版和新版公开接口，识别删除 API、函数签名变化（返回类型、参数增删改）、可见性收紧（`pub(all)` 变 `pub`）、struct 字段变化、trait 方法变化、enum variant 变化等破坏性变更，区分兼容新增，并给出 major / minor / patch 的 SemVer 升级建议。它面向 mooncakes.io 发布前检查、PR API 审查、团队 CI 守卫等场景，帮助 MoonBit 包作者在公开发布前发现可能影响下游用户的 API 变化。

## 项目方向与适用场景

本项目属于 MoonBit 工程基础设施与工具链方向。MoonBit 生态会出现越来越多可复用包，包作者需要在发布新版本前判断公开 API 是否发生破坏性变化，并据此决定版本号和发布策略。`moon_api_guard` 复用 `.mbti` 作为稳定的接口快照输入，将接口差异转化为可读报告、机器可读 JSON 和 CI 退出码信号。

典型适用场景：发布到 mooncakes.io 前检查公开 API 变化；在 pull request 中审查是否引入 breaking change；根据 API diff 判断 SemVer 版本号；在 GitHub Actions 或 GitLink CI 中阻止意外破坏下游用户的 API 变更。

## 已实现的核心功能

- `ApiItem`、`ApiChange`、`ApiReport` 核心数据结构，全部带文档注释。
- 解析 `pkg.generated.mbti` 中的函数、struct、enum、trait、type、typealias、fnalias、const、suberror、impl 声明，支持多行块和属性行。
- 细粒度破坏性变更分类：API 删除、返回类型变化、参数增删改、可见性收紧、字段增删/类型变化、trait 方法增删改、enum variant 删除。
- enum 仅新增 variant 判定为兼容变更。
- SemVer 建议（major / minor / patch）与 CI 阻断信号（release_blocked）。
- Markdown 报告（逐条列出变更）与 JSON 报告（含 changes 数组，字符串已转义）。
- CLI：`moon run cmd/main -- check old.mbti new.mbti`，破坏性变更时退出码为 1，用法错误/读文件失败为 2。
- 基线守卫工作流：`baseline/pkg.generated.mbti` + `.github/workflows/api-guard.yml`，PR 自动对比基线。
- 57 个测试（白盒 + 黑盒 + CLI），MoonBit CI 覆盖 check / test / fmt / info。
- GitHub 与 GitLink 双远程同步，README 与申报书同目录展示，mooncakes.io 发布元数据与指南已就绪。
