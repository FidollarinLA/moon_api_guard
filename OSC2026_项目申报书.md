## 2. 项目简介

`moon_api_guard` 是一个面向 MoonBit 生态的公共 API 兼容性守卫工具。项目基于 `moon info` 生成的 `.mbti` 接口描述，对比旧版和新版公开接口，识别删除 API、函数签名变化、类型变化、trait 变化、alias 变化等破坏性变更，区分兼容新增，并给出 SemVer 升级建议。它面向 mooncakes.io 发布前检查、PR API 审查、团队 CI 守卫和 OSC 项目验收前自查等场景，帮助 MoonBit 包作者在公开发布前发现可能影响下游用户的 API 变化。

当前仓库已经完成 MoonBit 项目骨架、README、Apache-2.0 License、核心数据结构、基础 API 对比逻辑、`.mbti` 简单 public 行解析、SemVer 建议、白盒测试和一个最小可运行示例。后续会继续补齐更完整的 `.mbti` 解析规则、报告输出、命令行入口和 CI 集成示例。

## 3. 项目方向与适用场景

本项目属于 MoonBit 工程基础设施与工具链方向。MoonBit 生态在发展过程中会出现越来越多可复用包，包作者需要在发布新版本前判断公开 API 是否发生破坏性变化，并据此决定版本号和发布策略。`moon_api_guard` 的目标不是替代 `moon info`，而是复用 `.mbti` 作为稳定的接口快照输入，将接口差异转化为可读报告、机器可读结果和 CI 信号。

典型适用场景包括：发布到 mooncakes.io 前检查公开 API 变化；在 pull request 中审查是否引入 breaking change；根据 API diff 判断 major、minor、patch 版本升级；在 GitHub Actions 或 GitLink CI 中阻止意外破坏下游用户的 API 变更；为 OSC 项目验收提供可复现的接口兼容性检查流程。

## 4. 拟实现的核心功能

- 提供 `ApiItem`、`ApiChange`、`ApiReport` 等核心数据结构，统一表示公开 API 项、兼容性发现和比较报告。
- 解析和规范化 `pkg.generated.mbti` 中的函数、类型、结构体、枚举、trait、impl 和 alias 等公开接口。
- 检测 API 删除、函数签名变化、类型定义变化、公开性收紧等破坏性变更。
- 检测新增 public API 等兼容变更，并区分 breaking / compatible / informational / unknown 等结果类别。
- 根据变更类型给出 `major`、`minor`、`patch` 的 SemVer 升级建议。
- 输出终端摘要、Markdown 报告和 JSON 报告，方便人工审查和 CI 系统消费。
- 提供 `moon_api_guard check old.mbti new.mbti` 命令，发现破坏性变更时返回非零退出码。
- 提供 README、最小可运行示例、核心测试、GitHub Actions / GitLink CI 模板，并在完成后发布到 mooncakes.io。

## 5. 原创 / 移植 / 参考说明

本项目为原创 MoonBit 项目，不直接移植特定开源项目，也不复制第三方源码。项目概念上参考 Rust `cargo-semver-checks`、Java `japicmp` / `revapi` 等 API 兼容性检查工具，但输入格式、规则分类、CI 集成方式和发布流程都围绕 MoonBit `.mbti`、`moon info` 与 mooncakes.io 重新设计。
