## 项目名称

moon_api_guard：MoonBit API 兼容性守卫工具

## 项目简介

moon_api_guard 是一个面向 MoonBit 生态的公共 API 兼容性守卫工具。它基于 `moon info` 生成的 `.mbti` 接口描述，对比旧版和新版公开接口，识别删除 API、函数签名变化、类型/trait/alias 变化等破坏性变更，区分兼容新增，并给出 SemVer 升级建议。项目服务于 mooncakes.io 发布前检查、PR API 审查、团队 CI 守卫和 OSC 项目验收前自查等场景，目标是形成一个真实可用、边界清晰、文档完整、可测试、可发布到 mooncakes.io 的工程基础工具包。

## 项目方向与适用场景

工程基础设施 / API 兼容性与发布守卫工具。适用于需要在 MoonBit 中维护公共 API 稳定性、审查 PR 破坏性变更、决定 SemVer 版本号、在 CI 中阻止意外 API breakage 的库作者、工具开发者和生态维护者。

## 拟实现的核心功能

1. `ApiItem`、`ApiChange`、`ApiReport` 等核心数据结构。
2. 解析和规范化 `pkg.generated.mbti` 中的函数、类型、结构体、枚举、trait、impl 和 alias。
3. 检测 API 删除、参数或返回类型变化、公开性收紧等破坏性变更。
4. 检测新增 public API 等兼容变更，并给出 major / minor / patch 的 SemVer 建议。
5. 输出终端摘要、Markdown 报告和 JSON 报告。
6. 提供 `moon_api_guard check old.mbti new.mbti` 命令。
7. 提供 README、可运行示例、核心测试、GitHub Actions / GitLink CI，并后续发布到 mooncakes.io。

## 原创 / 移植 / 参考说明

原创项目。项目采用 Apache-2.0 License，不直接移植特定开源项目，不复制第三方源码。概念上参考 Rust `cargo-semver-checks`、Java `japicmp` / `revapi` 等 API 兼容性检查工具，但实现对象、规则和输入格式针对 MoonBit `.mbti` 与 mooncakes.io 发布流程重新设计。当前公开查重未发现成熟同名或高度相似的 MoonBit 专用 API 兼容性守卫工具，但该结论不能绝对排除未公开项目。
